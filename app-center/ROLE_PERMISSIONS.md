# 角色功能权限表

本文档详细说明了系统管理员、企业管理员和员工三种角色在系统中的功能权限。

## 角色定义

| 角色 | 判断条件 | 说明 |
|------|---------|------|
| **系统管理员** | `owner === 'built-in'` 且 `is_admin === true` | 内建组织的管理员，拥有系统最高权限 |
| **企业管理员** | `owner !== 'built-in'` 且 `is_admin === true` | 企业组织的管理员，管理本企业内的用户 |
| **员工** | 其他情况 | 普通用户，只能访问应用中心 |

### 4. 用户管理（`/users`）

| 功能 | 系统管理员 | 企业管理员 | 员工 |
|------|:---------:|:---------:|:----:|
| 访问页面 | ✅ | ✅ | ❌ |
| 查看用户列表 | ✅（所有组织） | ✅（仅本组织） | ❌ |
| 创建用户 | ✅（可选择组织） | ✅（自动填充本组织） | ❌ |
| 编辑用户信息 | ✅（可修改组织、is_admin等） | ✅（仅本组织用户） | ❌ |
| 查看用户详情 | ✅ | ✅ | ❌ |
| 重置用户密码 | ✅ | ✅ | ❌ |
| 设置用户为管理员 | ✅（可设置任意组织） | ✅（仅本组织） | ❌ |

**说明：** 
- 系统管理员和企业管理员都可以访问用户管理页面
- **系统管理员**：
  - 可以查看和管理所有组织的用户
  - 创建用户时可以选择组织（通过下拉框选择）
  - 可以修改用户的 `owner`（所属组织）和 `is_admin`（是否管理员）字段
- **企业管理员**：
  - 只能查看和管理本组织的用户（后端自动过滤）
  - 创建用户时自动填充当前组织，无法选择其他组织
  - 可以设置本组织内的用户为管理员（`is_admin=true`）

## 权限判断代码示例

```javascript
import { isSystemAdmin, isOrgAdmin, isEmployee, getUserRole } from '../../utils/role';

// 判断是否为系统管理员
if (isSystemAdmin()) {
  // 系统管理员逻辑
}

// 判断是否为企业管理员
if (isOrgAdmin()) {
  // 企业管理员逻辑
}

// 判断是否为员工
if (isEmployee()) {
  // 员工逻辑
}

// 获取用户角色
const role = getUserRole(userInfo);
// 返回值：'system_admin' | 'org_admin' | 'employee' | null
```

## 注意事项

1. **角色判断依据**：
   - 系统通过 `owner` 和 `is_admin` 字段判断用户角色
   - `owner === 'built-in'` 且 `is_admin === true` → 系统管理员
   - `owner !== 'built-in'` 且 `is_admin === true` → 企业管理员
   - 其他情况 → 员工

2. **数据隔离**：
   - 企业管理员只能看到和管理本组织的用户
   - 后端 API 会根据当前登录用户的 `owner` 字段自动过滤数据

3. **权限控制**：
   - 前端通过路由保护（`RoleProtectedRoute`）和菜单显示控制权限
   - 后端 API 也会进行权限验证，确保数据安全

4. **员工角色**：
   - 目前员工角色只能访问应用中心
   - 未来可以根据业务需求扩展员工的功能权限
